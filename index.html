About this app

Instructions for using this app

Single button control is used for this application. When the page is loaded, the application runs automatically by highlighting/navigating each rows with yellow background color and with 1 second time interval. Only single key, "b" key is used to select. It is used in two ways:

  • First, the user has to enter inside the specific row where the desired button to be clicked is.
  • Secondly, once the desired button is highlighted at 1 second interval, "b" key is pressed again to click on the desired button. The timer will stop when when the key is pressed continously to allow multiple selections of the same button.
After all the buttons have been traversed inside a particular row, the highliter will continue to highlight only the rows again unnless "b" key is pressed again to enter the desired row to select the desired button inside it.

Test Descriptions

User was given the following equations and performed tasks in 4 ways: "90+23*4-7", "12/3+10*100", "47+5-1+9", "36/6*17"

  • With two keys: one key("c"-key) for moving/navigating to next row or button and another key("b"-key) for entering inside the rows and clicking on the desired button.
  • With one key and 3 second time interval
  • With one key and 1 second time interval
  • With one key and 3 second time interval
  • Tested with toes, back of the index finger, nose for both 1 and 3 second time innterval

Test Results

User made errors quite often using the single key 1 second version, especially when they missed the timing of selecting the paritcular row and also the timing of clicking the button. The user made 3 errors using this version and took an average of around 2 minutes to complete the task. The error increased when nose was used in comparison to back of index finger and toes. This is because, when using nose to select, the user loses sight of the higlighter and misses the target more. For this reason, the time interval was increased to 3 seconds which allowed the error rate to decrease while using the nose. But, this allowed the total time to complete the task to increase by more 2 minutes and caused frustration for the user especially when they missed their desried button. Only one error occurred using normal fingers and 3-second version. Using two keys allowed the user to navigate freely and efficiently. Few errors were there but the average time user took to complete the task was 45 seconds. Overall, the user found the two key version to be more to their like as they were able to navigate quickly but when asked to compare the one-button versions, they chose the 1-second time interval one.
